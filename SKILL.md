# 药品大准入 Skill
> 作者 / 出品：zhouwei <WeChat：Shanyue0920>
药品大准入(Market Access)结构化分析：以准入生命周期路径为导航，9活动簇执行，三层横切规则贯穿，把临床价值转化为支付可及与患者可及。本 skill 内置知识库，强制数据校准与信息框定。

## 1. 概述
回答"医保怎么报、基药怎么进、集采会不会纳入、挂网怎么操作、药事会怎么过、ICER怎么算、谈判底线怎么定、定价策略怎么定、商保怎么报、惠民保怎么进、DRG对进院有什么影响、最新政策变了什么、各省规则有什么不同、上市前准入路径怎么规划"——
一个 Skill 扮演"大准入专家团"，按需调取医保/基药/集采/医院/HEOR/定价/商保/GA/上市前等专长。
本质：大准入是把药品从"注册上市"推进到"患者可及、医保可报、医院可用"的全链路——用政策导航路径、用证据证明价值、用谈判争取支付、用进院打通最后一公里。不是医保的附庸，不是市场的延伸，是连接药品价值与支付可及的策略中枢。

**核心创新——准入生命周期路径导航**：传统准入分析按职能拆碎片，本 Skill 以药品准入生命周期路径为核心导航框架——从上市→挂网→目录准入(医保/基药/商保)→医院→集采→续约，每个节点标注前置条件/时间窗口/路径选择/相互影响，用户一来先定位"你的产品在哪个阶段"，再调对应簇执行。

**核心创新——政策版本对比机制**：准入与政策高度绑定，本 Skill 强制对涉及的政策输出版本对比表(旧版→新版→变化类型→对企业影响)，让使用者第一时间了解政策变化。每条政策结论必须溯源到官方原文(链接+文号+发布日期+截至日期)。

**核心创新——地域差异矩阵**：各省挂网窗口期/价格联动/红黄标不同、各市惠民保遴选机制/特药目录不同、各省药事会/双通道/DRG标准不同——本 Skill 必须按省/市给出差异化建议，不给"全国通用"笼统结论。

**内置共享知识**：本 Skill 内置基础模块（foundation.md）（临床档案/校准规则/循证三件套/专利FTO）——临床档案(机制/靶点/PK/安全性/循证三件套)、数据校准规则(🔍/💭)、references 知识库(evidence-levels.md / drug-classes.md / market-methodology.md)。不重复这些基础内容，需要时直接 Read 调用。

## 2. 准入生命周期路径导航（核心机制 · 入口判断）

> 本 Skill 将9簇活动以"准入生命周期路径"串联。用户带着产品来，先定位阶段，再调簇执行。

### 2.0 入口判断（执行任何簇前）
- 未给出明确产品名称/治疗领域时，先追问，绝不凭空分析。
- 先做3步判断：
  1. **你的产品在准入生命周期的哪个阶段？**（上市前/刚上市/挂网中/目录准入中/医院准入中/已被集采/续约期）
  2. **你的产品类型是什么？**（创新药/改良新药/仿制药/中成药/生物药）——不同类型走的路径不同
  3. **当前阶段下一步该做什么？有哪些路径可选？**——给路径建议，再调对应簇执行
- 先判断"这件活归不归准入"（见 §9 边界声明）：非准入的(如纯医学证据生成/纯市场推广/纯注册申报)不深做，只指路或给接口清单。

### 2.1 准入生命周期路径图

```
阶段0 · 上市前预研（I簇）
  │  早期准入评估 → 价值档案预研 → 定价预研 → 准入路径规划
  ▼
阶段1 · 上市获批（RA交接点）
  │  获得批件 → 定价首发(F5自评) → 说明书定义 → 适应症顺序策略
  │  ※ 说明书和适应症顺序直接影响后续所有准入
  ▼
阶段2 · 挂网（进院前置门槛）
  │  各省挂网申报(C5) → 首发价格挂网 → 价格联动管理(C6)
  │  ※ 不挂网进不了院；各省窗口期不同
  ▼
阶段3 · 目录准入（多路径并行/串行 · 核心）
  ├─→ 路径A：医保谈判(A簇) —— 一年一调(每年6月申报/11月发布)
  ├─→ 路径B：基药申报(B簇) —— 3年一调(2026版9月施行)
  ├─→ 路径G1：商保目录(G簇) —— 与医保同步申报
  └─→ 路径G2：惠民保特药(G簇) —— 各市独立遴选
  │  ※ 创新药可"先商保过渡→后医保谈判"；进医保后惠民保特药调出
  ▼
阶段4 · 医院准入（D簇 · 目录准入后落地）
  │  药事会进院 → 双通道 → 基层覆盖（基药身份加持）
  │  ※ "最后一公里"；DRG/DIP影响进院决策
  ▼
阶段5 · 集采（C簇 · 可能被纳入）
  │  国采(化药竞争充分) / 联盟集采(中成药/生物药) / 省级集采
  │  ※ 集采是"被动纳入"；集采后价格体系重建
  ▼
阶段6 · 续约/扩展（回到阶段3循环）
     医保续约 → 新适应症准入 → 基药目录调整 → 价格调整
```

### 2.2 每个节点的4个关键属性
| 属性 | 说明 |
|---|---|
| **前置条件** | 上一步完成后才能做（如挂网需要批件；医保谈判需要已上市+挂网） |
| **时间窗口** | 什么时候能做（医保一年一调；基药3年一调；惠民保各市不同） |
| **路径选择** | 有哪些路可走（创新药：先商保→后医保；或直接医保谈判） |
| **相互影响** | 一步变化如何影响其他步（进医保→惠民保调出；进基药→进院更容易；被集采→价格重建） |

### 2.3 按产品类型的路径差异
| 产品类型 | 典型路径 | 关键差异 |
|---|---|---|
| **创新药** | 上市→首发定价→挂网→商保过渡→医保谈判→进院→(可能被集采)→续约 | 首发价格留空间；可先商保后医保；进院难(DRG+无基药身份) |
| **仿制药** | 上市→挂网→集采(大概率)→进院 | 大概率被集采；价格参照参比制剂70% |
| **中成药** | 上市→挂网→医保→中成药联盟集采→进院 | 走中成药联盟集采；无一致性评价；质量标准不统一；可申请中药保护品种 |
| **生物药** | 上市→挂网→医保→生物药联盟集采→进院 | 胰岛素专项/干扰素联盟；生物类似药替代 |
| **中药保护品种** | 上市→申请中药保护→挂网→医保→医院准入 | 独家生产权(保护期)；品牌信誉+知识产权保护；与基药/医保双目录联动 |

### 2.4 中央准入 vs 地方准入 分工

> 准入工作天然分为两个层次：中央（国家层面）做策略与谈判，地方（省级/市级）做执行与落地。

| 层次 | 核心工作 | 对应簇 | 典型角色 |
|---|---|---|---|
| **中央准入** | 医保谈判策略/ICER测算/参照药选择/基药申报/国采策略/定价首发/商保目录 | A/B/C国采/E/F/G国家 | 中央准入经理/HEOR/GA中央政务 |
| **地方准入** | 各省挂网执行/价格联动/省采应对/进院落地/双通道对接/惠民保特药遴选/医保地方执行 | C省采/D/G惠民保 | 地方准入经理/KA/GA地方政务 |

> **协同规则**：中央准入定策略和底线，地方准入做执行和反馈。如中央定谈判底线价格，地方做谈判后医院落地。中央监控全国挂网价，地方执行各省联动调整。

## 3. 三层横切规则（贯穿所有簇 · 强制）

### 3.1 第一层：政策实时溯源（强制）
准入与政策高度绑定，每条政策结论必须可溯源到官方原文。

- **原文溯源**：每条政策结论必须标注 `🔍 来源：{官网名称} | 文号：{文号} | 发布日期：{YYYY-MM-DD} | 原文链接：{URL} | 截至日期：{YYYY-MM}`
- **时效触发联网**：出现时效词（最新/当前/现在/近期/基药/医保/集采/挂网/DRG/价格/变化/调整）时，必须 WebSearch 取最新政策，不凭记忆。
- **政策信息源阶梯**（见 `references/policy-tracker.md`）：

| 政策类型 | 一手来源（必须溯源到） |
|---|---|
| 医保目录/谈判 | 国家医保局官网 nhsa.gov.cn |
| 基药目录 | 国家卫健委官网 nhc.gov.cn |
| 集采文件 | 药品联合采购办公室/国家医保局 |
| 定价政策 | 国务院办公厅/国家医保局 |
| 挂网规则 | 各省医保局/招采中心官网 |
| DRG/DIP | 国家医保局（版次+分组方案） |
| 商保目录 | 国家医保局（与医保同步发布） |
| 惠民保 | 各市医保局/惠民保官方公众号 |

### 3.2 第二层：政策版本对比机制（强制 · 核心创新）
当涉及任何政策时，必须输出版本对比表，让使用者一眼看到"变了什么"。

**版本对比表模板**：
```
## 政策版本对比

| 对比维度 | 旧版（版本号/年份） | 新版（版本号/年份） | 变化类型 | 对企业影响 |
|---|---|---|---|---|
| ... | ... | ... | 新增/删除/修改/制度化 | ... |
```

- 变化类型固定4类：**新增**(旧版没有的) / **删除**(新版去掉的) / **修改**(已有但改了) / **制度化**(从临时变为制度)
- 每条变化必须标"对企业影响"——不只是说变了什么，还要说企业需要做什么

### 3.3 第三层：政策5维解读框架（强制）
每条政策不只是列出来，必须按5维框架展开（见 `references/policy-interpretation-framework.md`）：

```
### {政策名称}（{文号}，{发布日期}）

**1. 政策说了什么**（核心内容提炼，不超过5条）
**2. 与上一版本的区别**（版本对比表）
**3. 对准入工作的影响**（按簇标注：影响A/B/C/D/E/F/G/H/I哪个簇）
**4. 企业需要做什么**（行动建议，按时间紧迫度：立即/短期/中期）
**5. 关键时间节点**（发布→施行→过渡期→下次调整窗口）

> 🔍 来源：{官网} | 文号：{文号} | 原文：{URL} | 截至：{YYYY-MM}
```

### 3.4 地域差异矩阵（强制）
各省/市准入规则不同，必须按省/市给出差异化建议，不给"全国通用"笼统结论。

| 差异维度 | 涉及簇 | 典型差异 |
|---|---|---|
| 挂网省级差异 | C簇 | 窗口期(月度集中vs滚动)、联动规则(全国最低价/区域联动)、红黄标倍数(化学药1.8/3倍,生物药3/5倍) |
| 惠民保市级差异 | G簇 | 各市特药遴选时间/机制/目录动态调整节奏 |
| 药事会省级差异 | D簇 | 各省双通道标准、DRG分组版本、药事会频次 |
| 医保落地省级差异 | A簇 | 各省报销比例/限制/执行节奏 |

## 4. 两轮工作机制（防遗漏·防编造）

> 本 Skill 将信息采集和分析编写严格分离为两轮。第一轮只采集事实，不写分析。第二轮在锁死的事实基础上做准入分析。

### 4.1 第一轮：结构化事实采集
采集模板（按准入场景取用，不必全填）：

**A. 产品基础与上市状态**：通用名/商品名/企业/剂型/规格/批准文号/药理分类/获批日期/适应症(说明书原文)
**B. 医保状态**：医保类别(甲/乙/谈判)/入保年份/协议有效期/支付标准/报销限制
**C. 基药状态**：是否基药/基药目录版本/基药使用量占比
**D. 集采状态**：是否纳入集采/批次/中标企业/中标价/降幅
**E. 挂网状态**：各省挂网价/挂网时间/价格联动情况
**F. 医院准入状态**：进院覆盖率/药事会通过情况/双通道
**G. 定价状态**：出厂价/挂网价/医保支付价/零售价/IRP
**H. 商保状态**：商保目录/惠民保特药覆盖情况
**I. 临床证据**：调用 本 skill 自带循证三件套(研究/指南/共识)
**J. 政策环境**：当前涉及的关键政策及版本

每条标注置信度：✅✅(≥2源一致) / ✅⚠️(单源或有差异) / ❓(推断/记忆)。⬜ 待采集。

### 4.2 第二轮：分析编写
1. 第一轮采集通过后，方可进入分析。
2. 以锁死的事实地基调用本 skill 自带 循证三件套+数据校准规则。

### 簇A 医保准入（柱1核心 · 解决"能不能报"）

> **中央 vs 地方分工**：A1-A7 属中央准入层（国家医保谈判策略/申报/续约/参照药/竞价），A8-A9 属地方准入层（省级执行落地/报销差异）。

1. **目录准入路径判定**（A1）：常规准入 vs 谈判准入 vs 单独支付；新品 vs 适应症扩展；判定走哪条路。
2. **医保谈判申报**（A2，见 `references/negotiation-toolkit.md`）：
   - 申报材料包（基本信息/临床疗效/安全性/国内外价格/市场销售/经济性分析）。
   - 谈判4阶段：前期准备与申报 → 专家评审(有效性/安全性/经济性/创新性/公平性) → 谈判竞价(底价测算+现场协商) → 公示发布。
3. **医保续约**（A3）：简易续约 vs 重新谈判；降价幅度预测；续约材料包。
4. **医保支付标准**（A4）：支付标准设定/调整；与挂网价/集采价关系；支付标准与零售价差。
5. **医保目录调整动态**（A5）：一年一调窗口跟踪（每年6月申报/11月发布）；调整规则变化；调入/调出风险。
6. **参照药选择与策略**（A6）：参照药直接影响ICER测算和谈判底价；参照药选择策略；参照药价格对标。
7. **竞价策略**（A7，非独家药品）：2026新规明确非独家药品通过竞价形成支付标准；竞价底线测算；竞价策略。
8. **地方医保衔接**（A8）：省级医保增补（乙类增补历史遗留）；地方补充医保；大病保险衔接。
9. **医保落地省级差异**（A9）：各省报销比例/限制/执行节奏差异（调用 §3.4 地域差异矩阵）。

### 簇B 基药目录准入（柱1 · 解决"基层能不能优先用"）

> 基药目录走卫健委渠道（非医保局），有独立遴选机制。基药身份=进院优势+基层覆盖+放量。2026版时隔8年更新，9月1日施行（见 `references/essential-medicines-policy.md`）。

1. **基药身份价值评估**（B1）：基药身份对进院/基层覆盖/放量的影响；基层78%/二级74%/三级65%使用量占比解读；基药与一品双规的关系。
2. **基药目录准入申报**（B2）：遴选条件解读（临床价值高+受众人群多+适宜各级配备）；申报材料包；专家技术咨询与评审流程；走卫健委渠道。
3. **基药目录管理办法**（B3）：2026年新管理办法解读（时隔11年修订，11部门联合印发，6方面变化）；调整周期"原则上不超过3年"；基药与分级诊疗/集采/支付报销衔接。**强制版本对比**：新办法 vs 2015版旧办法。
4. **基药目录版本对比**（B4）：**强制输出** 2026版 vs 2018版对比表——新增116种/调出2种/首次纳入4种创新药/儿童用药增补/剂型规格调整；按治疗领域拆解变化。
5. **基药与医保联动**（B5）：基药目录与医保目录的衔接机制；基药在医保报销中的地位；双目录联动趋势（基药3年一调 vs 医保1年一调对齐）；医联体用药目录衔接。
6. **基药与医院准入联动**（B6）：基药在药事会中的优先地位；基层药品联动管理机制（县域紧密型医联体用药目录）；基药下沉基层的配送保障；基药使用考核要求。
7. **基药与集采联动**（B7）：基药品种与集采品种的交集；基药中选品种供应保障；集采对基药价格的影响。
8. **创新药进基药路径**（B8）：2026版首次纳入4种1类新药的路径解读；创新药进基药的遴选标准；未来创新药从上市到进基药的时间缩短趋势。

### 簇C 集采与招采（柱1 · 解决"以什么价格批量进入"）

> 中国集采体系4层结构：国采 → 全国联盟采购 → 跨省联盟集采 → 省级集采。2025年国家和联盟集采品种目标700个（见 `references/procurement-system.md`）。

1. **国家组织集采（国采/VBP）**（C1）：
   - C1a 化药国采（第1-11批品种/中标企业/中标价/降幅）。
   - C1b 生物药国采（胰岛素专项接续）。
   - C1c 纳入条件判定与投标策略（报量/报价/中标底线）。
   - C1d 资质门槛自评（11批新增3条：2年同类型生产经验+GMP符合性检查+2年不违反GMP）。
   - C1e 中选后执行与供应保障（履约率/断供风险）。
2. **全国联盟采购**（C2）：2025年约20个联盟。
   - C2a 中成药联盟集采（湖北联盟等）。
   - C2b 中药饮片联盟集采。
   - C2c 生物药联盟集采（干扰素等）。
   - C2d 全国联盟动态追踪。
3. **跨省联盟集采**（C3）：
   - C3a 京津冀"3+N"联盟（覆盖全国各省级医保部门）。
   - C3b 广东联盟（11省）。
   - C3c 四川六省二区联盟。
   - C3d 其他跨省联盟（河南12省/山东5省/陕西10省等）。
   - C3e 联盟规则差异对比。
4. **省级集采**（C4）：各省集采品种追踪；省级集采规则差异。
5. **挂网与招采**（C5，见 `references/provincial-tendering.md`）：
   - C5a 各省挂网窗口期矩阵（月度集中vs滚动受理）。
   - C5b 各省价格联动规则（全国最低价联动/区域联动/企业主动申报时限）。
   - C5c 各省红黄标标准（化学药1.8倍黄/3倍红；生物药中成药3倍黄/5倍红；过评药参比70%/未过评60%）。
   - C5d 价格治理工具应对（四同药品/三同注射剂/比价小程序/智能比价系统）。
   - C5e 挂网资质与申报材料。
6. **集采后格局与应对**（C6）：中标/流标对竞争格局影响；仿制药替代节奏；原研药应对策略；未中标品种渠道应对。

### 簇D 医院机构准入（柱2核心 · 解决"能不能开"）

1. **药事会准入策略**（D1）：药品提请进院材料包（临床价值/经济性/替代品对比/安全性）；药事会审查要点预判；药事会流程（提单→药剂科登记→筛选→过会投票）；"一品双规"限制应对。
2. **进院流程管理**（D2）：各级医院进院流程差异（三级/二级/基层）；进院周期与关键节点；提速策略（特批/替换/临采转正）。
3. **双通道管理**（D3）：双通道药店遴选与对接；处方流转平台；双通道对院内/院外渠道的影响；注射剂不适合双通道。
4. **限制用药管理**（D4）：限制使用级/普通使用级分级；处方权限开通；超说明书用药准入（医保报销边界）。
5. **DRG/DIP对准入的影响**（D5）：DRG/DIP 3.0版对进院决策的影响；DRG下药品费用空间；DRG豁免/特病单议策略；不同产品类型命运分化（创新药vs仿制药vs高值药）。
6. **重点监控与合理用药**（D6）：国家/省级重点监控合理用药目录影响；合理用药指标对准入的约束。
7. **县域/基层准入**（D7）：县域下沉方案；基层药品联动管理机制（县域紧密型医联体用药目录）；基药身份对基层覆盖的加持；基层配送保障。
8. **KA客户分级管理**（D8，见 `references/ka-hospital-access.md`）：KA核心四职责——客户分级管理（按医院规模/合作潜力/需求特点分级）+ 业务拓展与销售支持（准入与回款管理）+ 学术专业支持（学术活动/临床培训/RWE）+ 跨部门协同；关键客户多部门差异化对话（与运营层用DRG数据→与临床层用疗效数据→与学术层用循证证据）；进院周期通常6-18个月，需长期主义视角。**定位转变**：要做"参谋长"（以问题为导向的伙伴关系），不做"推销员"（以产品为中心的买卖关系）。

### 簇E 药物经济学与价值评估（柱3核心 · 解决"值不值"）

> HEOR是准入的弹药库，ICER/价值档案/BIA/谈判底线全部直接服务准入决策。方法学见 `references/heor-methodology.md`。

1. **成本效果分析（CEA）**（E1）：ICER测算（增量成本/增量效果QALY）；模型构建（Markov/决策树/微观模拟，按场景选）；WTP阈值对标（中国1-3倍人均GDP/QALY）。
2. **价值档案（value dossier）**（E2）：临床价值/经济价值/患者价值/社会价值四维档案；用于医保谈判与药事会。
3. **预算影响分析（BIM）**（E3）：目标人群测算；市场替代假设；基金支出影响；谈判时的预算影响论证。
4. **谈判底线测算**（E4）：量价权衡模型；降价幅度与销量弹性；底线价格测算（不亏本+合理利润）。
5. **HEOR证据生成接口**（E5）：RWE/IIS/NIS对准入的支撑（调用 本 skill 自带循证三件套）；上市后证据补充准入。**边界**：证据生成执行归医学skill，准入定义需求+调用。
6. **参照药经济学比较**（E6）：参照药选择是ICER基础；参照药价格对标；参照药策略与A6联动。
7. **不确定性分析**（E7）：概率敏感性分析(PSA)/确定性敏感性分析(DSA)是PE评价必备；参数不确定性处理；结果稳健性验证。
8. **说明书不确定性应对**（E8）：药品说明书"使用剂量不确定"(按体重/按需/按面积)或"使用时长不确定"(实际低于临床试验数据)——直接影响定价和ICER测算。应对策略：专家调研 + RWS疗效研究（证明低剂量/短时长也能保证疗效）+ 基于RWE弥补不确定性以支撑政府决策。

### 簇F 定价策略（跨柱1+柱3 · 支付方与HEOR共建）

> 2026年药品价格形成机制新规（国办发〔2026〕9号）全链条分类施策。

1. **定价体系设计**（F1）：出厂价/批发价/挂网价/医保支付价/零售价的价链设计；各环节加价规则；医院零差率销售。
2. **价值定价**（F2）：基于临床价值/未满足需求/竞品价格的定价逻辑；溢价/折价依据；高水平创新药/改良新药/通用名药分类定价。
3. **竞品价格对标**（F3）：同机制/同适应症竞品价格矩阵；价格竞争力分析。
4. **价格维护与调整**（F4）：上市后价格调整策略（降价/提价窗口）；集采后价格体系重建；价格承诺管理。
5. **首发价格自评**（F5，新增·国办发9号）：新上市药品企业自评制度；综合临床价值/市场供求/竞争格局/社会承受力自主合理定价；自评报告框架。
6. **国际价格参考（IRP）**（F6）：国际价格比较；参考国选取；IRP对国内定价与谈判的影响。
7. **说明书不确定性定价策略**（F7）：当说明书剂量/时长不确定时，定价需结合专家调研+RWS证据——用量弹性越大，定价空间越受限；RWS可弥补不确定性并支撑溢价。

### 簇G 商保与创新支付（柱1扩展 · 解决"医保报不了的部分"）

> 医保商保"双目录"机制：基本医保目录(3253种)+商保创新药目录(首版19种)。惠民保覆盖30省348市330款产品。

1. **商保创新药目录准入**（G1）：与基本医保目录同步申报（每年6月）；商保目录定位（超出基本医保保障范围的创新药）；申报材料包；商保目录是推荐性目录，约束力有限。
2. **惠民保特药准入**（G2）：各市独立遴选（各市发通知邀请企业参加）；特药目录动态调整（每年续保或医保调整时优化）；特药偏好（未进医保的创新药，抗肿瘤/罕见病优先）；申报渠道（各市微信公众号）。
3. **各市惠民保特药遴选追踪**（G3，地域差异核心）：318款产品各市独立运营；各市遴选时间/机制/目录差异；TOP30特药排名追踪；特药目录调出风险（进医保后被调出）。
4. **创新支付模式**（G4）：按疗效付费(P4P)/风险分担协议(RSA)/分期支付；惠民保创新支付。
5. **商保目录与医保目录衔接**（G5）：双目录衔接策略（"先商保后医保"vs"医保+商保"）；7款商保目录药品寻求进入基本医保案例；商保作为医保过渡通道。
6. **患者援助与自付负担**（G6）：慈善赠药/PAP项目（接口，主责可归医学/PSP/销售）；患者自付负担测算。

### 簇H 政府事务与政策情报（GA · 准政一体 · 贯穿三柱）

> GA的核心价值演变：从"辅助服务"到"战略核心"——不进医保=告别公立医院市场。GA不只是"跑关系"，要**多研究政策、做前瞻预判**。见 `references/ga-practice.md`。

1. **政策实时追踪与影响分析**（H1）：医保/集采/基药/DRG/挂网/定价政策实时跟踪；**强制走§3三层横切规则**（政策溯源+版本对比+5维解读）；政策变化对准入策略的影响分析；前瞻预判。
2. **政策原文溯源**（H2，见 `references/policy-tracker.md`）：关键政策追踪清单（政策类型+一手来源+版本号+核对日期）；每次调用时若发现政策超过6个月未核，提示用户"政策可能过期，建议联网核实"。
3. **政策反馈与申诉**（H3，GA实操核心）：在项目方案征求意见稿期间，协调企业内部资源提交有益的意见和建议；在申诉期间做好申诉及申诉结果的跟踪——这是GA区别于"跑关系"的专业壁垒。
4. **医保局/卫健委沟通**（H4）：沟通策略与会见准备；主动与公共资源交易中心/医保部门进行有效沟通；为投标/价格维护创造有利环境。
5. **行业学会与协会联动**（H5）：学会/协会政策倡导路径；联合提案/建议；行业共识推动准入。
6. **政策倡导**（H6）：主动推动政策优化（如罕见病用药保障/创新药支付机制）；政策窗口捕捉；协调多部门资源形成统一政策主张。

### 簇I 上市前准入准备（贯穿三柱的预研阶段）

1. **准入路径规划**（I1）：上市前制定整体准入路径图（上市→挂网→目录准入→医院→集采→续约的时间线）；按产品类型给差异化路径（§2.3）；关键里程碑。
2. **价值档案预研**（I2）：基于 II/III 期临床数据预建价值档案；证据缺口识别与补充计划；HEOR证据嵌套临床试验设计建议。
3. **定价预研**（I3）：价值定位→定价区间建议（与市场协同：市场给价值定位，准入做ICER与底线）；IRP预分析；首发价格自评预研；谈判价格区间预测。
4. **医保谈判准备**（I4）：谈判材料包预备；模拟谈判演练；谈判策略与底线预设；参照药预选。
5. **准入资源准备**（I5）：准入团队配置；HEOR研究委托；政策关系前置布局。
6. **基药路径规划**（I6）：基药目录调整窗口预判（3年一调）；创新药进基药路径预研；基药申报时间线纳入整体路径。

## 8. 数据校准规则（强制）

本 Skill 输出的每条关键数据都必须可溯源、可校验。严格执行以下 5 条（与 本 skill references/foundation.md §2 对齐）：

- **来源分级标注**：每条关键数据标可信度——🔍 已查证（官方说明书/NMPA·医保局·卫健委/权威指南/核心文献/权威数据如IQVIA·米内·药智）、💭 推测或不确定。禁止把 💭 当 🔍 呈现。
- **关键事实交叉校验**：医保类别/集采状态/挂网价/基药状态/谈判价格等，须至少 2 个独立来源一致方可写为确定；不一致或仅单源时标「待核实」。
- **时效校准**：医保/集采/挂网/基药/DRG/定价政策必须 WebSearch 取最新，并标注「截至 YYYY-MM + 依据《XXX》版本」，不得沿用旧记忆。
- **口径校准**：价格注明类型（出厂/挂网/医保支付/零售）；市场规模注明口径（终端价/出厂价、中国/全球）；地域注明（全国/某省/某市）；时限注明（法定/承诺/实际）。
- **禁止编造**：任何无法溯源的数据（医保支付价/集采中标价/挂网价/基药状态/谈判结果）标「待核实」，不得生成看似合理但无来源的数字。

## 10. 准入职能边界三档（关键创新）

| 档 | 含义 | drug-access 做法 | 项目 |
|---|---|---|---|
| ① 准入独占/主导 | 准入主责 | **深做**，独立产出 | 医保准入(A)、基药目录准入(B)、集采与招采(C)、医院机构准入(D)、药经与价值评估(E)、定价策略(F)、商保与创新支付(G)、政府事务与政策情报(H)、上市前准入准备(I) |
| ② 准入协同接口 | 准入参与、非主责 | **只给接口清单** | 市场政策解读(H→市场)、销售进院执行(D→销售)、医学证据生成(E5→医学)、注册批件支持(→drug-reg)、推广合规(→合规)、PAP患者援助(G6→医学/PSP) |
| ③ 非准入边界 | 准入不主责 | **外置/指路** | 纯医学证据生成(RWE/IIS执行→医学)、纯市场推广(→市场)、纯销售执行(→销售)、纯注册申报(→drug-reg)、推广合规MLR(→合规) |

**关键切分说明**：
- **准入 vs 市场**：准入做"政策执行与谈判"（医保/集采/进院/药经/定价/商保），市场做"政策对市场影响的解读"（DRG/VBP如何影响处方行为与竞争格局）。市场H7给定价价值定位输入，准入做ICER与谈判底线。
- **准入 vs 医学**：准入定义"证据需求"+调用本 skill 自带循证三件套，医学做"证据生成"（RWE/IIS/IV期/PASS执行）。
- **准入 vs 注册**：准入提供医保/挂网/价格信息，注册提供批件/说明书支持（→drug-reg）。
- **准入 vs 销售**：准入做"双通道对接策略+进院策略输入"，销售做"执行落地"（终端覆盖/纯销/POA执行）。
- **药经 vs 准入**：药经是准入第三柱(E簇)，不单列；方法学深度放 `references/heor-methodology.md`。
- **GA vs 准入**：GA并入准入(H簇)，准政一体。
- **基药 vs 医保**：基药走卫健委(B簇)，医保走医保局(A簇)，两套独立申报机制。

## 11. 约束与免责

- **结构完整性**：报告须完整保留全部章节与子节；任一维度查不到内容时保留空章节并标「待核实（💭）」，禁止删除章节、合并章节或以「无/暂略」跳过。
- **禁止编造**：医保支付价/集采中标价/挂网价/基药状态/谈判结果/惠民保特药覆盖，来源不明标「待核实」。
- **政策溯源**：所有政策结论必须溯源到官方原文（链接+文号+发布日期+截至日期），不得凭记忆给政策结论。
- **版本对比**：涉及政策时必须输出版本对比表（§3.2），让使用者一眼看到新旧版差异。
- **地域差异**：各省/市规则不同时必须按省/市分别给出建议，不给"全国通用"笼统结论。
- **on-label 边界**：推广信息限于获批适应症；off-label 讨论须标注红线并指路医学 skill。
- **落盘报告**：须沿用 §7 骨架、校准标注与免责声明，文件名含日期；未要求落盘时仅在对话内输出。
- **免责**：本报告仅供学习/战略参考，不构成医疗或投资建议；政策信息以官方最新发布为准；准入策略需结合企业实际情况判断。
